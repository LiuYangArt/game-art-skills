# Perforce P4 使用说明

这份说明是给团队成员看的，不需要懂技术。

## 这个工具是做什么的

它是给 AI agent 用来帮你处理 Perforce 的。

你可以用它做两类事情：

- `p4-init`: 第一次在新电脑上配置 P4
- 日常使用: 之后继续让 agent 帮你处理同步、workspace、分支、冲突等事情

如果你是新人，最常用的是 `p4-init`。

## `p4-init` 是什么用途

`p4-init` 的作用是让 agent 帮你完成第一次本地配置。

通常它会帮你做这些事：

- 检查电脑里有没有安装 P4
- 询问你的账号信息
- 询问你需要同步哪个项目或 stream
- 帮你确定本地文件放在哪里
- 帮你创建 workspace
- 需要的话，再帮你同步引擎
- 把本机后续要用的本地配置写好

你不用自己理解这些技术步骤，按 agent 提问回答就可以。

## 你应该怎么做

最简单的方式是直接把这些信息发给 agent：

- 说一句：`p4-init`
- 如果团队有 onboarding 文档，把文档链接或路径一起发给 agent
- 如果没有 onboarding 文档，再把服务器地址单独告诉 agent
- 你的 Perforce 用户名
- 你的 Perforce 密码
- 你的岗位或工种
  例如：程序、动画、角色美术、场景美术

如果你不知道自己该用哪个 stream，也没关系，直接告诉 agent 你的岗位，它会先给你推荐，再让你确认。

## `p4-connection.json` 是什么

这是你本机的连接配置文件。

它只给你本机使用，用来记录：

- 服务器地址
- 用户名
- 是否保存密码

这个文件是本地文件，不会默认上传到仓库。
模板里不再写死真实 server。

`workspace/client` 和 `charset` 不应该让用户手填。
正常情况下让 `p4` 按当前目录、`P4CONFIG`、`P4CLIENT` 或本机已有设置自动判断就可以。

## 怎么配置 JSON

大多数人其实不用手动配置。

更推荐的方式是：

1. 直接让 agent 帮你执行 `p4-init`
2. agent 在过程中帮你生成或更新这个 JSON

如果你确实需要手动填写：

1. 找到 `config` 文件夹
2. 复制 `p4-connection.template.json`
3. 把复制出来的文件命名成 `p4-connection.json`
4. 打开后补上这些字段：

- `server`: 服务器地址，从 onboarding 文档或负责人处获取
- `user`: 你的用户名
- `password`: 你的密码。如果团队不要求长期保存，可以留空

如果你不知道某一项该填什么，不要自己猜，直接问 agent 或负责人。

## 给团队同事的建议

- 新电脑第一次配置：优先用 `p4-init`
- 如果团队有 onboarding 文档：直接发给 agent
- 如果你不懂 stream、workspace、引擎分支：不要自己猜，直接让 agent 问你并推荐
- 如果你是美术或其他非技术岗位：只需要告诉 agent 你的岗位和你要做的内容即可

## 一句话理解

把 `p4-init` 理解成“让 agent 帮你把 P4 第一次装好并配好”；
以后需要继续用 P4 时，再让 agent 帮你处理就行。
